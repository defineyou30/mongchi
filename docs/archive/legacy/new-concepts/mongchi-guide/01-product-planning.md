# 01 Product Planning

## Product Promise

Mongchi lets users bring their beloved dog or cat into a tiny phone world where they can care for it, talk to it, decorate its home, and receive comfort.

## Target User

Primary:

- People who deeply love their dog or cat.
- Users who enjoy cozy games and personalization.
- Users who want a low-pressure companion experience.

Secondary:

- Users who like collectible decoration games.
- Users interested in AI-generated avatars.
- Users who may pay for deeper emotional interaction or premium personalization.

## Main Loops

### First Session Loop

```text
Welcome
-> Name and personality
-> Upload photo
-> Hatching generation
-> Pet reveal
-> First terrarium
-> First reaction
-> First care action
-> First reward
```

Goal:

- Convert curiosity into emotional ownership.

### Daily Loop

```text
Open app
-> Pet greets user
-> Care action
-> Reaction
-> Small reward
-> Optional decorate / chat / walk
```

Goal:

- Give value in 30 seconds.

### Long-Term Loop

```text
Return over days
-> Bond grows
-> New reactions unlock
-> Items accumulate
-> Terrarium changes
-> Premium chat or special treats deepen attachment
```

Goal:

- Build durable affection and collection goals.

## Product Stages

### Prototype

Purpose:

- Prove onboarding, pet reveal, home terrarium, and local reactions.

Must include:

- Fake/mock generation path.
- Static pet asset.
- Basic care state.
- Authored reactions.

### Alpha

Purpose:

- Prove real upload/generation and daily loop.

Must include:

- Real photo upload.
- Generation job.
- Pet reveal.
- Feed/talk/walk/affection/water.
- Basic inventory.

### Beta

Purpose:

- Prove retention, safety, and monetization signals.

Must include:

- Better reaction catalog.
- Generation retry.
- Privacy deletion.
- Premium chat gate.
- Analytics.

### Launch

Purpose:

- Public-ready iOS/Android release.

Must include:

- Store compliance.
- Purchase restore.
- Support.
- Monitoring.
- Security review.

### Live Ops

Purpose:

- Keep the world fresh.

Must include:

- Seasonal events.
- New items.
- Reaction expansion.
- Cost monitoring.
- User feedback loop.

## Success Metrics

Activation:

- Welcome to upload conversion.
- Upload to generation completion.
- Generation acceptance rate.
- First care action completion.

Retention:

- Day 1 return.
- Day 7 return.
- Care action frequency.
- Reaction engagement.

Monetization:

- Premium chat gate view.
- Premium chat conversion.
- Item pack conversion.
- Regeneration purchase conversion.

Quality:

- Generation failure rate.
- User rejection reason.
- Chat moderation rate.
- Support tickets by category.

## Open Product Decisions

- Brand name and tone.
- Dog/cat only vs more species.
- One main pet vs multiple pets.
- Free retry count.
- Premium chat model: subscription, credits, or bundle.
- Currency model: none, soft, premium, or mixed.
- Launch languages and countries.
- Whether step/GPS walking ever enters product.
