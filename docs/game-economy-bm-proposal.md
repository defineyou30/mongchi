## 0. 설계 원칙 (힐링앱 톤 유지)

1. **광고는 유저가 선택하는 리워드형만.** 전면/배너 광고 금지 — 힐링 경험을 깨는 순간 리텐션 근간이 무너진다.
2. **결제는 '더 잘 돌보기 위해'가 아니라 '표현과 편의'.** 돈을 안 쓰면 펫이 불행해지는 구조(죄책감 과금) 금지. 무료 루프만으로 펫은 항상 행복해질 수 있어야 한다.
3. **모든 유료 요소는 대사/연출로 보상.** 아이템을 사면 펫이 특별하게 반응하는 것 자체가 상품 가치.

## 1. 아이템 특수효과 → 시간제 버프 시스템 ★핵심

**현재:** `specialItemEffectsByAction` — 사용 즉시 스탯 보너스뿐. 아이템 간 차별성이 약하고 "특별한 효과" 체감이 없음.

**제안:** 시간제 버프(activeBuffs) 도입.

| 버프 종류 | 예시 아이템 | 효과 |
|---|---|---|
| 감쇠 감소 | 든든한 스튜 | 4시간 동안 포만감 감쇠 −50% |
| 획득 증폭 | 새 장난감 (3회 내구) | 놀이 행복 획득 +50% |
| 산책 강화 | 레인코트 | 비/눈 오는 날 산책 보상 2배 + 전용 대사 |
| 회복 증폭 | 라벤더 쿠션 | 휴식 기력 회복 +50%, 수면 전용 대사 |
| 유대 부스트 | 추억의 담요 | 2시간 유대 XP 2배 |
| 날씨 보호 | 미니 파라솔 | 더운 날 행복 감쇠 무시 |

**구현 메모:**
- `Inventory` 또는 `CareState`에 `activeBuffs: Array<{ buffId, kind, magnitude, usesLeft?, expiresAt, sourceItemId }>`
- `projectCareStateForTime`(감쇠 계열)과 `applyLocalCareAction`(획득 계열)에서 modifier 적용
- HUD 게이지 아래 활성 버프 아이콘 + 남은 시간
- 버프 발동/만료 시 전용 반응 규칙 (엔진 `eventContext` 활용: `buff_started`, `buff_expired`)

**BM 연결:** 소모품 = 크레딧의 반복 소비처. 리워드 광고로 랜덤 버프 1회 획득 가능.

## 2. 산책 리워크 + 리워드 광고 ★BM 1순위

**현재:** mock 15초 즉시 복귀. 기다림도, 마중의 설렘도 없음.

**제안:**
1. **실제 비동기 산책 (15~30분).** 돌아오면 로컬 푸시 "몽치가 돌아왔어요! 마중 나가주세요."
2. **"광고 보고 마중 나가기"** — 리워드 광고 시청 → 산책 즉시 완료(타임 스킵). 다마고치류에서 검증된 최고 효율 광고 배치.
3. **"간식 들려 보내기"** — 간식 아이템 소모 or 광고 시청 → 산책 보상 2배.
4. **산책 일기** — 돌아올 때 2~3줄 미니 스토리("작은 새를 만나서 한참 인사했어", 날씨/계절 연동). 반응 엔진 규칙으로 저작, 수십 개 풀.
5. **수집 도감** — 산책 수집품(계절 잎사귀, 반짝이 조각, 깃털, 조약돌…)을 도감에 등록. 시즌별 완성 보상(크레딧+한정 장식). `walk_return_rare` 카테고리와 연결. 수집욕 = 무과금 리텐션의 뼈대.

**광고 밸런스 가드:** 일일 리워드 광고 캡 3~5회. "광고를 봐야만 하는" 구조가 아니라 "보면 조금 이득"까지만.

## 3. 본드 레벨 보상 트랙

**현재:** XP만 쌓이고 레벨업해도 아무 일 없음 (`getBondLevelFromXp`).

**제안:** 레벨 보상 테이블.

| 레벨 | 보상 |
|---|---|
| 2 | 새 대사 팩 해금 (성격 심화 대사 10줄) |
| 3 | 무료 채팅 티켓 +2 |
| 4 | 전용 장식 아이템 1종 |
| 5 | 특별 애니메이션(celebrate 변형) + 마일스톤 연출 이벤트 |
| 7 | 대사 팩 2차 (커플티어: 유저를 부르는 애칭 등장) |
| 10 | 한정 테마 배경 + "단짝" 칭호 |

- 레벨업 순간 = 감동 연출 이벤트(펫이 먼저 다가와서 특별 대사) — 힐링앱의 "정서적 과금 포인트" 직전 경험.
- (2차) 시즌 케어 패스: 30일 무료/프리미엄 트랙, 케어 행동으로 패스 XP. 프리미엄 트랙에 한정 장식/대사 팩.

## 4. 소품·식물 배치 문제 해결 ★유저 페인포인트

**문제 진단:** 자유 배치는 (a) 좌표 조작 UX가 모바일에서 어렵고, (b) 소품 아트가 배경 라이팅/원근과 안 맞아 "스티커 붙인 느낌"이 남.

**해법 — 자유 배치를 버리고 3단 접근:**

1. **소켓(자리) 방식 확정** — 이미 `HomePlacementLane` 15종 + `defaultPlacementByLane`이 있음. 이걸 밀고 나가서: 배경(테마)마다 소켓 좌표 세트를 정의하고, 유저 선택을 "어디에 둘까"(어려움)에서 **"이 자리에 뭘 둘까"**(쉬움)로 뒤집는다. 빈 소켓은 은은하게 반짝여서 채우고 싶게 만든다. 동물의숲이 아니라 다마고치식이 몽치 톤에 맞음.
2. **시각 통합 규칙 (스티커 느낌 제거)**
   - 바닥 접촉 그림자를 아이템 이미지에 굽지 않고 **앱에서 공통 그림자 스프라이트로 합성** (홈 펫 스프라이트의 `homePetShadow`와 동일 패턴)
   - 배경별 **색 보정 틴트 오버레이**: 밤 배경이면 소품에 푸른 틴트, 노을이면 주황 틴트 (React Native `tintColor`/반투명 오버레이로 저비용 구현)
   - 스케일 표준화: `sceneSize` 필드가 이미 있으므로 소켓별 허용 크기 강제
3. **테마 번들로 전환 (BM 겸용)** — 개별 소품 단품 판매 대신 **"봄 정원 세트"** 같은 배경+소품 코디 번들. 배경과 소품을 같은 프롬프트 배치로 함께 생성하므로 미스매치가 원천 차단됨. `selectedTerrariumThemeId`가 이미 존재. 번들은 단품보다 객단가도 높다.

## 5. BM 전체 구조 (수익원 정리)

| 수익원 | 내용 | 상태 |
|---|---|---|
| 리워드 광고 | 산책 스킵·보상 2배·티켓 +1·랜덤 버프 (일일 캡 3~5회) | 신규 |
| 크레딧 소액과금 | 간식·버프 소모품, 장식 단품 | 소비처 확장 |
| 테마 번들 | 배경+소품 세트 (중가, ₩3,300~5,500) | 신규 |
| Plus 구독 | 무제한 힐링챗 + 월간 테마 + 상시 버프 1슬롯 + 광고 제거 | 채팅 기존, 혜택 확장 |
| 시즌 패스 | 무료/프리미엄 케어 패스 | 2차 |

**단위 경제 메모:** 생성 원가 ~$0.08/유저(시트+low 적용 후) 기준, 리워드 광고 eCPM $10~20 가정 시 광고 3회 시청 유저 1명이 하루 $0.03~0.06 → 생성 원가는 이틀 내 회수. 구독/번들은 순마진.
