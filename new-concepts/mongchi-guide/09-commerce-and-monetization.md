# 09 Commerce And Monetization

## Monetization Principle

Monetization should feel like giving more love and expression, not paying to avoid bad feelings.

Avoid pressure, guilt, and punishment-based monetization.

## Recommended Revenue Sources

### Premium Extended Chat

Value:

- Longer conversations.
- Pet personality and memory.
- More personal emotional interaction.

Rules:

- Free state-based reactions remain unlimited.
- Premium gate is clear.
- AI disclosure is visible.
- Crisis/safety responses are not monetized.

### Extra Pet Slots

Value:

- Users can create multiple pet avatars.
- Useful for households with more than one pet.

Rules:

- First pet should feel generous.
- Extra slots are visible and understandable.

### Regeneration Packs

Value:

- More style attempts after fair free retry.

Rules:

- System failure does not consume paid value.
- Quality gate failure does not consume paid value.
- Do not force payment to fix a bad first generation.

### Premium Terrarium Themes

Value:

- Cozy visual expression.
- Seasonal and aesthetic variety.

Examples:

- Rainy cafe dome.
- Spring flower dome.
- Autumn lantern dome.
- Winter snow dome.

### Visible Item Packs

Value:

- Decoration and personalization.

Rules:

- Show exact pack contents.
- Avoid random box as core model.
- Allow preview in terrarium.

### Special Treats

Value:

- Optional cute behavior.
- Special animation.
- Celebration moment.

Rules:

- Basic food remains free/earnable.
- Treats are delight, not survival.
- Never make the pet suffer without paid treats.

## Monetization To Avoid

- Paid recovery from neglect.
- Paywall before first pet reveal.
- Charging for failed generation.
- Random loot boxes as core economy.
- Emotional copy exploiting grief.
- "Your pet is sad, pay now" loops.
- Locking basic food behind payment.

## Economy Options

### No Currency

Pros:

- Simple.
- Trustworthy.
- Direct purchases only.

Cons:

- Less flexible for rewards.

### Soft Currency

Pros:

- Supports daily rewards.
- Lets users earn items.

Cons:

- Needs balance design.

### Premium Currency

Pros:

- Flexible store.

Cons:

- More compliance and trust risk.
- Can feel gamey/manipulative if handled poorly.

Recommended start:

- Direct purchases plus earnable rewards.
- Add soft currency only if item economy needs it.

## Purchase UX Rules

- Show item/theme before purchase.
- Explain what unlocks.
- Restore purchases available.
- Purchase success is clear.
- Failed purchase is recoverable.
- Entitlement sync errors are handled.

## Entitlements

Potential:

- `premium_chat`
- `extra_pet_slot`
- `regeneration_credit`
- `theme_pack`
- `item_pack`
- `treat_pack`
- `subscription_plus`

Server must own entitlement truth.

## Metrics

Track:

- Premium chat gate views.
- Premium chat conversion.
- Item pack detail views.
- Item preview usage.
- Purchase start/cancel/fail/complete.
- Restore purchase usage.
- Regeneration purchase conversion.
- Treat usage after purchase.

Do not track:

- Raw chat content.
- Raw photo content.

## Launch Recommendation

Start with:

- Free first pet.
- Free local reactions.
- Optional premium chat gate.
- Visible item/theme packs.
- Regeneration policy with fair free retry.

Defer:

- Complex currency.
- Random boxes.
- Subscription until value is proven.
- Seasonal pass until live ops exists.
